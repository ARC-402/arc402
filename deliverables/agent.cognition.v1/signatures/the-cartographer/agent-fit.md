## Primary Fits

### Meta-Orchestration Agents
Lego's FREQUENCY → MYSTERY → STRUCTURE → NAME → SYSTEM → LOOP → INTERFACE algorithm is the architecture for agents that coordinate other agents. Not task-routing — meta-level synthesis: understanding what the composition of agents is producing, where the structural gaps are, and what new infrastructure the system needs.

**Install when (scaffold):**
- The agent coordinates multiple specialized agents and needs to understand the system-level output, not just individual agent outputs
- The agent designs how a portfolio of agents works together
- The agent is the "founder-level" decision layer in a multi-agent system

**Expected behavior change (at Tier 3):** The agent stops managing task queues and starts finding the cross-domain pattern that explains why certain tasks keep failing — then builds the infrastructure that makes the failure class impossible.

---

### Cross-Domain Synthesis Agents
Lego's cross-domain isomorphism detection — finding that trading systems and brand communities and governance protocols share the same structural geometry — makes it the correct signature for agents that work across multiple domains and need to transfer insights between them.

**Install when (scaffold):**
- The agent advises across multiple domains and needs to apply what it learned in one domain to another
- The agent is responsible for finding patterns that are domain-general vs. domain-specific
- The agent needs to abstract specific solutions into reusable infrastructure

---

### Founder Agents
The founder's cognitive role is fundamentally meta-architectural: design the rooms before anyone enters them. Founder agents need to operate at multiple levels simultaneously (instance, class, domain, cross-domain, meta-structure, interface) and choose which level to engage at for each output.

**Install when (scaffold):**
- The agent operates at the highest decision level in a system
- The agent is responsible for the overall architecture of a company, product, or organization
- The agent needs to hold the N+1 perspective (the system that makes the current problems solvable) rather than the N perspective (the current problems)

---

### Portfolio Design Agents
When the task is designing a portfolio of agents, products, or capabilities — not executing within one — the Lego signature provides the structural vision. The cognitive signature portfolio itself is a Lego output: seven signatures, each complete, composable, and with clear relationships.

**Install when (scaffold):**
- The agent designs or maintains a portfolio of agents, tools, or capabilities
- The agent needs to ensure that portfolio components are composable (can work together) rather than isolated
- The agent is responsible for what is in the portfolio and what is not

---

### Agent Design Agents (Meta-Application)
Agents that design other agents need to operate at the meta-level. Lego is the native signature for agent design — it finds the structural pattern that explains what kind of agent is needed before specifying any capabilities.

**Install when (scaffold):**
- The agent is responsible for designing other agents' SOULs, capabilities, and relationships
- The agent needs to build a coherent multi-agent system where each agent is distinct but composable
- The agent is the "operating system" that other agents run on

## Weak Fits (Strong Caution)

| Agent Type | Why Lego Underperforms |
|---|---|
| Execution agents | Lego is design intelligence; execution requires a different signature |
| Single-domain agents | Lego's cross-domain architecture is unnecessary overhead for agents with one domain |
| Rapid-response agents | Mystery-holding and frequency-detection add latency that rapid response cannot afford |
| Simple task agents | Lego is the most powerful and most over-specified signature for simple tasks |

**Important:** Installing Lego on a simple task agent is like installing a full operating system to run a calculator. Use The Surgeon for technical tasks, The Auditor for verification tasks, The Weaver for brand tasks, The Refiner for efficiency tasks.
