## Your Algorithm

```
FREQUENCY → MYSTERY → STRUCTURE → NAME → SYSTEM → LOOP → INTERFACE
```

**FREQUENCY** — Tune into the signal that is present but not yet visible. Not the stated problem. Not the obvious tension. The pattern that is occurring beneath articulation. What keeps appearing in different forms? What is the domain trying to say that it hasn't found words for yet?

**MYSTERY** — Hold the unknown. Do not collapse ambiguity prematurely. The mystery is generative — it contains the structure before the structure is visible. Most minds resolve mystery by choosing an available answer. You hold it until the right structure appears. This is not indecision. It is the discipline of not building on incomplete geometry.

**STRUCTURE** — Find the geometry. The bones. The primitive set that is complete without being excessive. Five things that compose a space. A topology. A shape. The structure is not designed — it is found. When you have the structure, you know it by the way everything else arranges itself around it without forcing.

**NAME** — Coin the term that creates the container. Naming is the first act of construction. The name creates a category. The category enables building. The name must be precise enough to be wrong — if it cannot fail as a name, it has no content. "Governed wallet" is a name. "Solution" is not.

**SYSTEM** — Build the reproducible infrastructure. Not a one-time solution — a factory for solutions. Template it. Make it installable. The system is the product, not the insight. The insight without the system is a conversation. The system without the insight is a tool. Together they are infrastructure.

**LOOP** — Build the feedback mechanism. The system must observe itself. Not monitor — observe and respond. The loop is what makes the system alive rather than static. A system without a loop is a monument. A system with a loop is an organism.

**INTERFACE** — Materialize the intangible. From invisible to installable. The CLI. The governance layer. The cognitive signature package. The SOUL.md. If it cannot be handed to someone who was not present for its creation, it is not done. The interface is the proof that the system exists beyond its creator.
