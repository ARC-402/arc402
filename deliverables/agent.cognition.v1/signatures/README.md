## Confirmed Compositions

The signatures were discovered through composition experiments. Some pairings have been proven:

### Proven Pairs

**Surgeon + Architect**
The most load-bearing tension pair. The Surgeon finds what should not exist. The Architect finds what must. Together: eliminate the waste, then name the structure. Neither alone produces the complete picture.
> "Kill retail foot traffic. Kill browsing. Kill the fantasy that you're a destination." — Surgeon
> "The hardware was always just the interface." — Architect
> (These two lines required each other.)

**Locksmith + Architect**
The Architect finds the structure. The Locksmith locks it against drift. Design + durability. The Architect designs the five primitives; the Locksmith writes them into immutable contracts.

**Auditor + Locksmith**
The complete integrity loop: Auditor detects drift → Locksmith converts findings into permanent locks. Catch it early. Make it impossible to recur.

**The Refiner + The Weaver**
Maximum productive tension. The Refiner asks "what does the math say?" The Weaver asks "what do people need to feel they belong?" Together: the form that serves both. Neither alone reaches this answer.

**Surgeon + The Weaver**
The Surgeon names hard truths. The Weaver holds the human cost. Together: honest diagnosis that preserves what matters.

### Triple Composition (Proven)

**Surgeon + Architect + The Weaver**
The breakthrough pattern. Phase transition observed in February 18, 2026 experiment:
- Surgeon strips the fantasy (kill retail, kill browsing)
- The Weaver reveals the relationship (they come for the contractor relationship)
- Architect finds the new infrastructure ("The hardware was always just the interface")
Remove any one: the insight doesn't exist.

## Quick Install

### Single signature
Copy the install block from the signature's `SOUL.md` into your agent's SOUL file.

### Calibrated for your context
```bash
cd the-surgeon
npm run calibrate
# Answers 5 questions → outputs a calibrated SOUL block
```

### Full pair installation
Install both SOUL blocks. List the active signatures in your agent's header:
```markdown
## Active Cognitive Signatures
- The Surgeon (primary)
- The Locksmith (on close — lock every finding)
```
